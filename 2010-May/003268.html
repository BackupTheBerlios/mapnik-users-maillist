On 20-5-2010 15:30, Richard Weait wrote:

> then something like this to make my-os.xml
> |generate_xml.py osm.xml --dbname gis --accept-none > my-osm.xml|
>
> Edit generate_tiles.py near line 184 to find your my-osm.xml

It's not necessary and usually not even recommended to give osm.xml as 
an argument and to save to another stylesheet file.

If you omit osm.xml, generate_xml.py will only write 3 include files in 
inc/. For any case where you do not want to keep multiple OSM styesheets 
with different settings around, this is all you need.

An OSM stylesheet set up this way can be kept up to date by performing 
an svn up. Only if there are major changes to the inc/*.template files, 
would you have to rerun generate_xml.py.

The other advantage is that you do not have to edit the other scripts to 
point to your custome stylesheet.
