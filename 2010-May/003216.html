The nik2img 0.5.x series is out.

$ easy_install -U nik2img

Nothing too remarkable, except for one new option:

--mapnik-version

Basically, if you have mapnik2 (current svn trunk) installed as well  
as a previous mapnik version, nik2img will default to using mapnik2.

BTW, mapnik2 is experimental, so I recommend running Mapnik 0.7.1  
unless you want to experiment. :)

But, say you wish to run an XML file through both of them.

You can now do:

$ nik2img.py map.xml image.png --mapnik-version 1 # will internally do  
 >>> import mapnik

or

$ upgrade_map_xml.py map.xml > upgraded_map.xml # upgrade to mapnik2  
XML syntax
$ nik2img.py upgraded_map.xml image.png --mapnik-version 2 # will do  
 >>> import mapnik2

or like before:

$ nik2img.py map.xml image.png # will try mapnik2, then fall back to  
mapnik

Enjoy,

Dane
