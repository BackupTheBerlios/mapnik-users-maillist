El Viernes, 21 de Agosto de 2009, alain vicet escribió:
> Is it a known problem, I don't see anything about this on the mapnik
> users list?
> is it possible to reduce as much as possible these kind of truncated
> street names?

The things that can be done in order to reduce this are:
- Increase the size of the bbox (add a buffer) when rendering a zone, so you 
fetch more data, so that data creates new collisions that you couldn't see 
otherwise.
- Render bigger areas at once (this saves rendering a buffer that would be 
discarded) - this is already being done by OSM by using mod_tile and 
metatiles (groups of 8x8 tiles are rendered). Bigger metatiles would slow the 
rendering process (due to re-rendering non-changed data), but reduce the 
occurrences of truncated text linesymbolizers.
- Completely prevent linesymbolizers from rendering at the edges of the 
metatile. This could be achieved either by code, or by rendering first a fake 
text linesymbolizer all around the bbox of the metatile.
