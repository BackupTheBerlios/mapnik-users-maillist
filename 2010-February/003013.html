First getting started I recommend sticking the the merc/900913 default  
projection that osm2pgsql uses to import the data and which the  
stylesheets assume.

Often blank maps indicate a projection problem, and in the case of  
PostGIS, prior to Mapnik 0.7.0 could be resulting from failing SQL  
queries sent by Mapnik to PostGIS that went unnoticed (without obvious  
error).

Are you running Mapnik 0.7.0? If not I'd recommend upgrading which may  
help you slightly to understand how to get things set up correctly.

Dane
