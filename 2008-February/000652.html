Hi mapnik list, I'm a UMN Mapserver user but Mapnik looks very
interesting to me because it pretty.

However, Mapserver has one nice feature for interacting with ginormous
shapefile datasets (such as the entire TIGER set with is > 12Gb): a
2-level spatial index ("TileIndex"), where the first level indexes
entire data sets by extent, and refers to them by file path. The second
level is the normal per-dataset index. This is useful for splitting up
road data by state and county and makes quick whole-country map
generation feasible.

My question is, does mapnik have anything like this feature? Or is it
fast enough not to matter, for generating map images on the fly?

Thanks,
Dave
