That's fairly normal as running lwpostgis.sql to setup PostGIS does
not populate the spatial_ref_sys table.

You should have a spatial_ref_sys.sql file that will populate
it (it's in /usr/share/pgsql/contrib on Fedora) and if you run
that script against the database it should populate it.

Note that my one at least does not have 900913, the spherical
mercator projection used by Google, OSM, etc. If you need an
entry for that I have one.
