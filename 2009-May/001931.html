On May 9, 2009, at 2:30 PM, Marcin Rudowski wrote:

> Also if You really need smaller tiles, reduce number of used colors in
> Your styles, use less complicated icons and reduce number of displayed
> objects per Zoom. There is trade off: colorful and detailed map or
> bigger files.
>
> What mapnik really lacks, is some api for passing options to writers.
> For starters quality compression for jpeg and number of colors in  
> png256
> or some switch for transparency support in png256 (off, binary,
> multilevel) that I'm experimenting now.

FWIW, I've been handling this by rendering to instances of  
mapnik.Image, and then passing the results off to PIL for more control  
over image features. Looks like this:

	img = mapnik.Image(w, h)
	mapnik.render(map, img)
	img = PIL.Image.fromstring('RGBA', (w, h), img.tostring())

This may be a case where mapnik doesn't require the full set of  
writing options, perhaps.
