On 9 Jan 2008, at 21:27, Paul Smith wrote:

> On Jan 9, 2008 4:41 AM, Artem Pavlenko <artem at mapnik.org> wrote:
>> The best way to deal with text labels on short linestrings is to pre-
>> processe them (i.e. based on being adjacent and having common name)
>> into multilinestring. label_placement has been re-worked quite a lot
>> to deal with exactly this situation.
>> If you're storing your data in Postgresql you can use
>> 'multiple_geometries'  (true/false) parameter which controls how
>> labeling handled internally. Combining with 'spacing' you should be
>> able to achieve good results.
>
> I'm leaning that direction, but won't that result in having only a
> single label on a street? For street maps it's obviously useful to
> repeat labels every so often to orient users. Unless I'm mistaken from
> my brief tests and examination of src/placement_finder.cpp, each
> feature gets one and only one label. Correct?

You can have repeated text labels along line geometries. Key  
parameters are 'min_distance'
and 'spacing' , i.e try min_distance = 60 spacing = 100.

I have some examples to show you, but I need to ask first - I'm not  
sure if data is commercial or not.
