> Hi David,
>
> RE: http://trac.mapnik.org/ticket/60
>
> I'm concerned about removing the width and height attributes. I'm
> currently running some complex layering that requires transparent
> images to be rendered on one layer to reserve space for an icon in an
> overlay. So I just use a single 1x1 "transparent.png" and set the
> width and height appropriately for the size I want, and it saves me
> having to have multiple files e.g. "transparent16x19.png",
> "transparent18x18.png" etc.
>
> I would agree that the parameters shouldn't be required by default
> since most of the time you will want to render the image without
> scaling, but I think they should be optional for the (few) occasions
> where you may want to override the image's natural attributes.
>
> If the intention was to leave them available as optional, then I'm
> sorry for misunderstanding your proposal.

My original idea was to be able to 'cut' a label from one big image  
with lots of them:)
Obviously there are x0,y0 parameters missing in current implementation.
I think having x0,y0,height and width as optional parameters will  
work for all cases.
+1
