By default you get 32-bit rgba png and there are not optimal for web  
delivery.  Latest Mapnik can do (fast) quantization to palette based  
PNG which are much much smaller but the algorithm I'm using at the  
moment might produce undesired results . I have some ideas how to  
improve it so it's a work in progress.

As an alternative approach you can use PIL (Imaging) or ImageMagick  
(convert) or anything else to reduce colors , which would lead to  
smaller images.

For vector graphics, fast-fourier transform based compressions (jpeg)  
are no good.
