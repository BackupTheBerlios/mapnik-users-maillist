Hi,

I have an annoying issue when using estimate_extent=true on the postgis
input. I'm using the openstreetmap planet and unfortunately much of the data
is northern hemisphere so postgis incorrectly estimates the extent of the
layer which causes all the australian/south african data to disappear. So to
fix this I set estimate_extent=false, but this just causes mapnik to request
the full extent, and postgis seems to do this with a very lengthy table scan
which means mapnik takes ages to start rendering. This is slightly
irritating as I know the extent is pretty much the whole planet and there
won't be any performance gained by not querying the data source.

Is it possible to manually set the extent at all, either on the postgis
input or the layer itself? Or simply tell mapnik not to bother checking the
extent?
Or is there a much better way to handle this I haven't thought of?!

Thanks,

Dave
