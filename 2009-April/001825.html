Hi!


I have a strange rendering problem with mapnik. This concerns ways with 
the tag boundary=administrative. I have a rule using an image with a 
line-dot pattern for them:

<Rule>
<Filter>[boundary]='administrative'</Filter>
<MaxScaleDenominator>8806400</MaxScaleDenominator>
<LinePatternSymbolizer allow_overlap="true" file = 
"d:\Maps\Icons\way_border_web.png" type="png" width="32" height="1" />
</Rule>

This used to work nicely for many boundaries while I was rendering other 
parts of Germany.

In recent renders of the coast I noted that the ways are rendered
differently, sometimes even in the same area. Some are drawn correctly 
according to the above rule and some are drawn in the style defined for 
highway=primary. I can find no difference in the tagging.

For example here 
http://opentiles.com/nop/?zoom=13&lat=53.74541&lon=7.90323&layers=BT
- Way 29415285 around the island is drawn properly in a dotted line.
- Way 29412956 on the coast below is drawn as a red road

Any ideas what is going wrong here?
