On Apr 19, 2009, at 2:39 PM, David Leaver wrote:

> Is this meant to work with the WMS using XML styles?
> I tried it soon before 0.6.0 and it didn't seem to work, although  
> maybe I was doing something wrong.
>


Good catch. No. The way the OGC Server works is to parse out the  
layers and styles from the xml mapfile. Parsing out (and most notably,  
carrying through and applying the 'buffer_size' for each GetMap  
request) is not yet supported:

http://trac.mapnik.org/ticket/305

Also, the correct approach when using TileCache is to set the  
TileCache metabuffer:

http://tilecache.org/docs/README#configuration

>> Is there a way I can prevent this?
>>
>> Presuming you're on 0.6.0, change
>>
>> <Map srs="+proj=lcc +datum=NAD83" bgcolor="steelblue">
>>
>> to
>>
>> <Map srs="+proj=lcc +datum=NAD83" bgcolor="steelblue"  
>> buffer_size="128">
>>
>> Or some other value instead of 128 that's large enough to make your  
>> labels uncut. This will make mapnik fetch and render a larger bbox  
>> for your tiles, and then discard this gutter area to create your  
>> tile.
