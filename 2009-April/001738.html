I installed the new version of mapnik on Windows XP and it works neatly.


A few things that might be added to the installation guide:

- Mapnik must go into C:\mapnik_0_6_0\ as it has hardcoded dependencies 
to this folder.

- The installation guide I used for 0.5.1 instructed you to copy the 
site-package into the python installation rather than extend the path. 
It might be a good idea to delete this old copy when installing 0.6.0 to 
avoid confusion


And about running mapnik: I was able to render with my existing style 
right away. It appers that mapnik is _almost_ backwards compatible, the 
handling of font metrics and/or positioning in PointSymbolizer and 
Textsymbolizer seems to work a little bit different, at least my labels 
are all two pixels off to the right and bottom. (see attachments). But I 
guess it is a good idea to replace those self-made shields with regular 
shields now that they should work for points.


And finally a question: The release page mentions "Map buffers to avoid 
cut labels". Currently I have been implementing a buffer myself when 
rendering. Is that superfluous now? And where would I find more 
information on how to utilize them?


great work!

	Nop
