On Sat, 2007-11-10 at 11:02 +0100, Martijn van Oosterhout wrote:
> Hi,
> 
> I've come to mapnik from the openstreetmap project and there is a a
> mapnik XML map file in SVN people can use. However, within the mapfile
> there are lots of lines like:
>       <PointSymbolizer file="/home/mapnik/mapnik/symbols/peak.png"
> type="png" width="8" height="8" />
> With complete paths. And because everybody who runs mapnik will run it
> from a different places, all these paths need to be updated. In
> addition the database connection details are listed 9 times.
> 
> What I would like to suggest is that at the beginning of the file you
> can say something like:
>    <FileSource name="base">/home/mapnik</FileSource>
> And in the rules themselves:
> <PointSymbolizer source="base" file="symbols/peak.png" type="png"
> width="8" height="8" />
> 
> Similarly for the datasources something like:
> <DataSource name="db">
>   <Parameter ...>
> </DataSource>
> at the top level, and then in the layers:
> <DataSource base="db">
>   ... only parameters that are different ..
> </DataSource>
> 
> I beleive this would make it much easier to share config files as the
> information that needs to be customised can be escapsulated to a small
> part near the beginning of the file.
> 
> What do people think?

I think this is a great idea, and I had considered doing it myself.  The
other thing that would be nice is the ability to split the single XML
file into multiple parts and use xinclude to pull it all together.  I
meant to try this and see is the XML parser supports it, but I never got
round to it.

Keith.
