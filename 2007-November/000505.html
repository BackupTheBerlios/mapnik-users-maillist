> > </DataSource>
> > at the top level, and then in the layers:
> > <DataSource base="db">
> >   ... only parameters that are different ..

> I think this is a great idea, and I had considered doing it myself.

In that light, here's a patch that implements one of the ideas. I
basically copied the Datasource parser to the top-level and had it put
sets of parameters into a std::map. Later when parsing actual
datasources it looks for a base="foo" tag and if found uses that set
to initialise the list.

It works in my tests, but I've never done C++/STL before so I'd
appreciate it if you could look at it.
