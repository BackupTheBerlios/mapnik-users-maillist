Hi,

Jon Burgess wrote:
> The general solution is to set estimated_extent to false and set the
> extent to include all your data, or -180,-90,180,90 if you have the
> whole world.

[...]

> The one exception to this occurs if your rendering your map to some
> projection where the extent is either invalid or wraps.

Which is of course true if you render Spherical Mercator tiles; Mapnik 
will silently fail because it ignores the proj return code etc etc 
(details in http://trac.mapnik.org/ticket/308).

Use <Parameter name="extent">-180,-85,180,85</Parameter> to be on the 
safe side for 900913.
