Hi,

Great to hear from you. Thanks very much for your kind comments about 
mapnik. It can hopefully improve a lot as long as people post feature 
requests/ideas, like you have.
> First of all, mapnik is great and produces lovely stuff. I have some 
> requests/questions about text rendering and filtering though.
>
> 1. Labelling along line features, which is typically required for maps 
> with roads on :)
>   
Yes, I really want this feature too:).  Vince Spader is working on this 
at the moment. I'll see if we can combine efforts to get this done 
quicker. 
> 3. Spacing of labelling, which was mentioned in a thread "Textlabels" by 
> Jan Rademaker in May. It would be fantastic to be able to specify a 
> minimum distance in pixels before we the same label is repeated.
>   
This is a good feature,too. From Jan's posting:
" ...
What I would love to see is an option to specify the minimal distance
(in pixels, radius) between two labels. Probably more or less similar
to the collision detection but with a variable radius. That way, for
example, a river would always be identified by 4 labels on a 800x600px
map, regardless of the extent..."

This sounds the way to start and it won't require many changes. 
> 3. Wrapping labels so they are contained within a polygon. Say you have 
> a shopping center which is labelled "very very long feature name". It 
> would be nice to be able to wrap the label so it ended up centered in 
> the polygon and looking something like:
>
> very very long
> feature name
>   
I can see two approaches here:
1. Always wrap text if it is too long ( based on width/height ratio?).
2. For polygons, try to find a best fit (can be expensive).
Any more ideas?

> 4. Is there a way to filter based on the feature size? For example, only 
> displaying rivers that are longer than 1km at certain zoom levels. Is 
> this something worthwhile to add?
>   
I had plans to add support for 'functions' in filter expressions. AREA 
and LENGTH are good candidates. Having said that, pre-calculated values 
stored in the attribute would be more efficient.     
> If I get motivated enough I might start working on some of these myself, 
> but I'd like to get some feedback and ideas first.
>   
Absolutely. Does the above spark any more thoughts for you? Let's keep 
talking, since I'm really keen for more people to join us.

Cheers,
Artem
