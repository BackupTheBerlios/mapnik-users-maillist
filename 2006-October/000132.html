Yes! Sooner the better!
>> 3. Spacing of labelling, which was mentioned in a thread "Textlabels" 
>> by Jan Rademaker in May. It would be fantastic to be able to specify 
>> a minimum distance in pixels before we the same label is repeated.   
> This is a good feature,too. From Jan's posting:
> " ...
> What I would love to see is an option to specify the minimal distance
> (in pixels, radius) between two labels. Probably more or less similar
> to the collision detection but with a variable radius. That way, for
> example, a river would always be identified by 4 labels on a 800x600px
> map, regardless of the extent..."
>
> This sounds the way to start and it won't require many changes.
I guess there are two things. One, repeating of labels for the same 
feature. Second is repeating of identical labels for different features. 
I know one can probably work around this using nasty DB hacks but my 
data models (esp for road networks) come split into small segments :)
>> 3. Wrapping labels so they are contained within a polygon. Say you 
>> have a shopping center which is labelled "very very long feature 
>> name". It would be nice to be able to wrap the label so it ended up 
>> centered in the polygon and looking something like:
>>
>> very very long
>> feature name  
> I can see two approaches here:
> 1. Always wrap text if it is too long ( based on width/height ratio?).
> 2. For polygons, try to find a best fit (can be expensive).
> Any more ideas?
I think #1 is a good start, then wait and see if anyone requires a lot 
more cleverness than that (Pareto's Principle).
