I'm trying to add contour lines to the tiles I generate with mapnik using the
method described  http://wiki.openstreetmap.org/wiki/Contours in the
openstreetmap wiki  (using postgres method), but no contour lines appear.

This is running in a VM under ubuntu 10.04 server setup following 
http://weait.com/content/build-your-own-openstreetmap-server this info  ,
and with mapnik 0.7.1 setup as 
http://trac.mapnik.org/wiki/UbuntuInstallationOld here .

I've setup the database with just UK data and with srtm files for the same
area (I believe!).  I get straight tiles for various levels (I've done
levels 4 - 15), but the contour lines are not there.....

There are 1/2 million records in the contours table, so the data has landed.

The only thing I've found that might indicate the problem is that the way
column in the OSM imported data all start something like

010200002031BF0D0002000000F6285C6F2F2245C1B81E854BE113574185EB51984E

 (all the same up to BF0D00 for quite a lot of records)

whereas the contour table they all start

0105000000010000000102.

I don't know if this is relevant or not, I've not been able to find any info
on the syntax of this column.

Any ideas what's gone wrong?

Oh yes, I've looked at the database with qgis as well (very
sloooowwwwwwwwly) and the same result - roads etc. OK,no contours.

Also I built a debug version of mapnik and I could see it processing the
srtm contour layers in the debug output, but nothing to indicate why nothing
appears.  The srtm files I downloaded are the rectangle N49 to N60 and W10
to E0.
