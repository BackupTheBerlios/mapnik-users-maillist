>> "elevation" or other things like that.)  Is that what Mapnik calls
>> properties (props) on a feature?  If so, how can I add fields to

> Have a look in plugins/input/postgisfs.cpp for example :
> ....
> boost::put(*feature,name,val);  // val can be int, double, bool,
> UnicodeString  see <mapnik/value.hpp>
> ...
> Note that mapnik uses unicode strings exclusively, so for unicode strings:
> UnicodeString ustr = tr_->transcode(buf);
> boost::put(*feature,name,ustr);
