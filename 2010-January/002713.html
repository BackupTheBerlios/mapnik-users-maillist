Hi Chris,

I don't think you did anything wrong. My guess is that nobody has  
tested building boost via macports on 10.4(tiger). Before you give up  
I would try installing the latest XCode for 10.4. Apple makes this  
hard as you have to dig for anything but the latest version for the  
latest operating system, but it will be there on the ADN downloads area.

In general installing Mapnik dependencies with macports is only the  
easiest route if nothing goes wrong and the hardest route to debug if  
something does. For example, I'm not sure what is going on with that  
error you pasted - it does not provide much error info. You should  
still file a ticket at the Macports trac site with that error. I doubt  
many of the developers are still running 10.4 and would be able to  
test, but maybe they might have ideas.

The next option I recommend would be  installing boost from source. If  
you run into problems doing this then pasting the output back to this  
list would be fine.

Dane
