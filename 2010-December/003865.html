Ah, try this:

	$ cascadenik-compile.py cascadeniktest.mml output.xml

The reason we changed it in version 1.0 from a pipe-to-file to an explicit argument is that there is a bunch of path magic now happening in Cascadenik that's designed to make it more resilient to relative directory paths and to take advantage of a bunch of work Dane did to the Mapnik core.

Used to be that these three commands would all result in identical output files:
	
	$ cascadenik-compile.py cascadeniktest.mml > output.xml
	$ cascadenik-compile.py cascadeniktest.mml > .../output.xml
	$ cascadenik-compile.py cascadeniktest.mml > /tmp/output.xml

Now, these may result in slightly different output files:

	$ cascadenik-compile.py cascadeniktest.mml output.xml
	$ cascadenik-compile.py cascadeniktest.mml .../output.xml
	$ cascadenik-compile.py cascadeniktest.mml /tmp/output.xml

Hope that helps.
