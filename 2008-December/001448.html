I made a build of the current source code as of Friday with shape file
support that I thought I would make available in case it is of use to other
win32 users.  I also just did a quick build of the GDAL (1.4.1) module but
did not test it.  It only includes the mapnik.dll and .lib (no python
bindings), built with Visual Studio 2005.  I applied the patch as described
in Ticket #87 to address the shieldsymbolizer issues with points.  I
dynamically linked to the supporting libraries so the zip file includes
those dlls also.

 

Let me know if anyone else is interested in updated win32 builds.  I can
spend more time on this if there is an interest.  I also have a .Net C++
wrapper that lets you make calls into the mapnik.dll from .Net. I could
spend some time on that to make available if desired.
