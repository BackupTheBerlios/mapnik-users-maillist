This could be done in Mapnik trunk by turning the 'height' parameter into an "expression" in the C++ code.

I could help you with this patch if you are interested in testing it.

So with that functionality in place could then either do:

height="10"

or

height="[height]" # to pull from a field value at runtime

See http://mapnik.org/news/2009/dec/08/future_mapnik2/ for more details on this new feature.
