Hi,

I converted OSM data in the database into epsg:32635, edited osm.xml to use this projection, removed the extent lines from osm.xml and run the following command:

C:\mapnik>c:\python2.5\python nik2img.py -m nik2img_noextent.xml -o niktest2.png
  -p epsg:32635 -r 391000,6673000,396000,6678000 -s 5000,5000 -v

I wrote corresponding world file niktest2.pgw as

 1.0000
 0.0000
 0.0000
 -1.0000
 391000
 6678000

Mapnik map is full of OSM vector data and it can be overlaid with other maps and aerial photos from WMS service.  I am satisfied, finally I can use OSM maps in GIS.  I think that some day I will prepare a set of OSM maps which are ready to be used with OziExplorer moving map software.  Thanks for the developers.
