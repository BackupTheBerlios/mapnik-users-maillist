Marco Cagnoli wrote:

> I'm using mapnik in combination with tilecache to render off line a map.
> My question is about mapnik performance: what seems strange to me is 
> that I have a 100% usage for cpu while the memory allocation is very low 
> (20 - 30 MB).

Well if you're doing something CPU intensive then 100% CPU usage is a 
good thing. I'm also not sure why you expect CPU usage and memory usage 
to correlate in any way - they are entirely separate things and how much 
of each resource a program uses will depend on what it is doing and how 
it does it.

> My idea is that mapnik can't load into memory the data files and that 
> it's reading from the hard disk: it would explain the high cpu usage and 
> the not so good performance.

Quite the opposite - if you were waiting for disk I/O then you wouldn't 
have 100% CPU usage. The fact that you do tells you that he I/O system 
is able to provide data fast enough to keep the CPU busy.
