On Nov 7, 2008, at 7:05 AM, GeoTux wrote:

> Hi Dane, thanks by your attention, the 'Map server dependency' item  
> refers to if Mapnik works with any WMS service, or if only works  
> with one particular map server (e.g. GeoServer).
>

So Mapnik is a renderer but various WMS implementations can be easily  
built on top of it. Currently Mod_tile, TileCache, and the Built-in  
Mapnik OGCServer are the primarily servers that work with Mapnik.  
Mapnik is not a web application that would depend on a map server or  
need to work with any other mapserver, rather it is the backend that a  
mapserver would depend upon.


> The 'Require privative plug-ins' item refers to if the user have to  
> install some plug-in on browser to view web application normally  
> (e.g. there are some applications that only run with Flash plug-in).
>

No, Mapnik outputs png,jpeg,svg,pdf, and postscript and if these are  
loaded within a web application they will not be a factor that would  
require any additional plugins.
