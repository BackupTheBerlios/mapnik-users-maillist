I've been making improvements to the Mapnik SCons built system in trunk.

I've started a new wiki page explaining some the features and general  
gochas:

http://trac.mapnik.org/wiki/UsingScons

I'm looking for anyone who can test on various platforms and report  
problems they encounter, so i can rid the system of any bugs before  
the upcoming 0.6.0 release. Please post tickets to http://trac.mapnik.org 
  using the component 'SCons Build Scripts'.

One notable change is that users now have access to a 'configure'  
stage that, if successful, will cache all options and require no  
custom paths in order to then build and install mapnik. This should  
allow the SCons process to feel a bit more like the common ./configure  
&& make dance. Also available is the ability to set custom options in  
a 'config.py' file which is also described in the above wiki page.

Cheers,
