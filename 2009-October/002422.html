Beware, just fixed a bug that affects Mapnik (when using shapefiles)  
on Ubuntu Karmic and (I assume) any systems with g++ >= 4.4.1.

http://trac.mapnik.org/ticket/436

So, spread the word that folks should run trunk for the time being if  
upgrading to the latest debian systems. The Karmic release is slated  
for two weeks from now, Oct 29, and any Karmic users that install  
Mapnik 0.6.1 from source will hit this, although hopefully the  
recently accepted mapnik package will include this fix and be usable.

Thanks to Dominic for first notifying me of this problem.
