I upgraded my Ubuntu from Feisty to Gutsy yesterday. I also decided to 
clean up my stack of manually installed GIS apps. It solved the issue 
above, so I think my libiconv problem was due to a mess in my 
configuration rather than a mapnik bug (unless some changes were made 
regarding this issue between revision 552 and 555).

During the building process, an error occurred due to a missing Boost 
serialization include file. Solved by installing the 
libboost-serialization APT package.

I also found a workaround to install python modules in the directory 
specified in the PREFIX build option: in bindings/python/SConscript, I 
changed PYTHON_PREFIX by PREFIX at line 26. I can find mapnik in 
PREFIX/lib/python2.5/site-packages/mapnik. Of course, I had to change my 
Python path, I did it by adding sitecutomize module in the main 
site-packages directory. I don't know if this practice is good or bad 
but it works quite well.
