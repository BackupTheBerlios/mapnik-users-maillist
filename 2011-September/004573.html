I have a difficult challenge and have been totally flummoxed by it. I'm
wondering whether anyone else knows how to help.

I have been trying to create a projection where you define an Origin and the
projected distance from the origin is adjusted logarithmically but the angle
is kept the same.

If the actual distance is D, the projected distance* P=log(1+D/C) *where C
is a constant.

The actual distance and angle can be found using the Haversine formulae as
described here http://www.movable-type.co.uk/scripts/latlong.html and these
can then be used with the formula above to find x-y coordinate positions in
a bitmap image.

So far so good.

BUT the problem is that with the way this allows you to cover an enormous
distance in your projected, it means that different information is relevant
at different distances. Eg. in the immediate vicinity of the Origin,
footpaths are useful, whereas 300km away only country boundaries, land and
water are relevant. Likewise for text labels.

Also, road widths etc need to be adjusted according to their distance from
the Origin, so that they are large at close range and relatively smaller
further away.

I have got completely stuck. Does anyone have any idea how to achieve this??

The attached image of the South Bank in London, is an example of the output
using images downloaded from the OpenStreetMap site.

Thanks in advance for any hints!
