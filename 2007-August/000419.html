> 1) There's a Mapnik viewer available??? Such utilities would be great!

Yes, it's in trunk/mapnik/demo/viewer already. It requires qt4 (I use 4.3) and
you'll need to modify viewer.pro to match your system. Bear in mind this is a
work in progress.

>
> 2) From the title bar, it seems like you use an xml config file. In
> early posts I remember some people mentionned the load_map() function
> used with an XML file. As there's no demo, tutorial or sample available,
> I thought it has been phased out. Could you provide some materials about
> this? A reference or even the XML schema would be great but if you don't
> have time for this, some sample xml files would already help a lot.

I used to have RelaxNG schema somewhere ... but things moved on since then. I
think best places to look :
1. $(mapnik_src)/src/load_map.cpp - it references all parameters
2. http://trac.openstreetmap.org/browser/applications/rendering/mapnik/osm.xml -
is probably most comprehensive example of *.xml config. It is also rather
overwhelming:D

My idea to eventually create a map authoring tool which can read/write *.xml
config files and edit layers/styles/rule/filters/symbolizers :
http://artem.dev.openstreetmap.org/files/viewer.png


>As far as I remember, the style definition in such file follow the SLD
> specs, is that true?
Well, originally I was following SLD but it was too constraining and ,frankly,
not fit for purpose. I decided to free mapnik from it. I'd like to think of it
as SLD++.

One of the reasons I'm holding back on schema etc. is I see things are
evolving.. Have you got experience in this area?

Is there some specification for all the elements or
> only the style?
>

Do you mean, can all elements be referenced in *.xml? Yes, layers, data sources
and styles (including rules,filters, symbolizers etc).
