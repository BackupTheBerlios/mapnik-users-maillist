Hi,

My interpretation is that this years benchmark was a failure, because Linux machines with 8 GB of memory proved, surprisingly, to be able to cache data so effectively that some of the WMS servers actually were performing the tests without reading from disks at all. This was possible because even if the test run included 2000 individual WMS calls with moving bounding boxes and variable output image sizes, the subsequent runs were exactly the same. Some of the servers could not utilise in-memory cache as well and they were reading data from disk. Disk I/O was limiting the speed to around 20% of the from-memory rendering. But all-from-memory is not really a normal use case, and results were like comparing apples with oranges and therefore some teams did not want to publish the results, because people would just read the images/minute output as the ultimate result. I do not know if this PostGIS test was telling about some real differrences as well, perhaps it did. In all the other, obviously rather meaningless tests Mapserver on Linux did best numbers.
