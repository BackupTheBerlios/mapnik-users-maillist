On 8-9-2010 13:56, Buddy Mash wrote:

> Many thanks for your suggestion. This was indeed the problem. Lower and
> mixed-case TTF file names work, but an all-uppercase file name means the
> font gets ignored!

If the filename is all uppercase and fits in the 8.3 format (a SFN, 
Short Filename), then a LFN (Long Filename) will not be generated and 
stored by Windows. So it would seem that mapnik (or boost) only operates 
on files which have a LFN stored.
