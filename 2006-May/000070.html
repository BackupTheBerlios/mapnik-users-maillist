Denis,

Mapnik does not yet support cartographic projections.  As such, there is 
not datum per say, or any kind of projection for that matter.

You simply have to use underlying data in a unique projection, and use 
that coordinate system ... Mapnik is unaware of what it is though.

You can use lats and longs to your liking, assuming the underlying data 
is also in geographic coordinates.

If you're using the Python API, it includes a binding to the PROJ.4 
cartographic projections library: http://proj.maptools.org/.

 >> from mapnik import Projection

You could write your code using this to translate input/output 
information into the underlying coordinate system.

J.F.
