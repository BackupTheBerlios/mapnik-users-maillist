> in the hope that I?ll be able to use it to create beautiful maps from

Yes :) As we've discussed, very few in the mapnik community know or use MS
SQL server, but certainly this should be able to work. The task will be to
make sure that, when we can produce the Windows binaries (artem is working
on this), that the OGR version linked in has proper support for the latest
MS SQL server client library.

> I know that a Windows build system was one of the intended GSOC projects
this year but I understand that it ended up not reaching completion.

Correct. This has meant that we did not have time/money/energy planned for
an alternative solution, but Artem is trying to find time to work on this
and is making progress.

>  Is there anything that humble mapnik users can do to expedite a 
> Windows compiled binary release? (Other that, perhaps, wait 
> patiently?!)

For this push, I think just rooting Artem on is all that is needed, and
patience. In the future, to ensure timely binaries for windows I think
ideally we could fund someone to do it. Because none of the Core mapnik
developers are Windows users, we need to find help and key individuals or
companies that need windows support could help make that happen. Send an
off-list email if you want to follow up about this.
