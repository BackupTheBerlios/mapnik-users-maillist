Hi Mark,

On 25 Apr 2007, at 23:46, Mark Coletti wrote:

> I was puzzled why I was still getting the Boost::lexical_cast  
> exceptions when passing in {hix,hiy,lox,loy} parameters for  
> mapnik.Raster().  I figured out that the boost::lexical_cast<>  
> means that the floating point parameters should be passed in as  
> strings.  So this works:
>
> layer.datasource = mapnik.Raster( file='yell_15sep99_bands432.tif',  
> lox='487755.812', loy='4886394.5', hix='593235.812 ', hiy='4994874.5')
>

> Surely there's a better way to just pass in real values without  
> having to use strings?

I'm agree. I was thinking about re-implementing parameters as variant  
types for a while now.
> Or, for that matter, just have the underlying code figure out the  
> bounding box from the data itself?

Not in all cases. Some raster have no metadata and no world file. I  
think it's handy to be able manually geocode raster image.
But yes, when there's metadata we should use it.

> I'm a big believer in comments and am disturbed that there are NO  
> COMMENTS.  (Except for the obvious copyright and licensing  
> header.)  I'm wondering if these are stripped out before distribution?

No.
>   It'd help me a lot to extend code functionality and otherwise fix  
> any encountered problems.
>

I'm sorry to disappoint you with the lack of comments. It's my style  
I guess:).  If you want to help to add comments/improve/hack Mapnik  
you're very welcome.

I've committed prototype GDAL raster plug-in (latest trunk). It is  
still work in progress.

Give it a try: ds = mapnik.Datasource(type='gdal', file = '/path/to/ 
file')
