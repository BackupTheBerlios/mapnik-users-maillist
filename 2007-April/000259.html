We recently discovered that mapnik-0.4 is not using the shapeindex for 
shapefiles of type point. I discovered this when trying to get polygons 
to label properly. I was getting river labels way off of the actual 
rivers because they were being placed on the feature's centroid (not 
necessarily inside the feature). As a workaround, I used ESRI software 
to create centroids with forced placement inside features. When I tried 
to label the water features using the points datasource it took 120 
seconds to render (versus 3 seconds using the former polygon 
datasource). Has anyone else run into this issue? For now I'm converting 
all my points data into polyons as a workaround.
