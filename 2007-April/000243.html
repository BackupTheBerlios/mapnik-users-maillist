For raster datasource you need to provide  :
file -- full path to filename
format e.g 'tiff'
lox,loy,hix,hiy left-bottom and top-right corners defining bbox
Also, for tiff images it is important to strip or tile them ( with  
tiffcp ..)
One day I have almost written gdal plug-in for mapnik, which supports  
more rasters and can read geo-metadata embedded in GTiff or/and world  
files. I'll try to find time to finish it. Is this something you can
