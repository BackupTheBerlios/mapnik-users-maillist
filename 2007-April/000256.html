I was puzzled why I was still getting the Boost::lexical_cast exceptions
when passing in {hix,hiy,lox,loy} parameters for mapnik.Raster().  I figured
out that the boost::lexical_cast<> means that the floating point parameters
should be passed in as strings.  So this works:

layer.datasource = mapnik.Raster( file='yell_15sep99_bands432.tif', lox='
487755.812', loy='4886394.5', hix='593235.812', hiy='4994874.5')

Surely there's a better way to just pass in real values without having to
use strings?  Or, for that matter, just have the underlying code figure out
the bounding box from the data itself?

I'm a big believer in comments and am disturbed that there are NO COMMENTS.
(Except for the obvious copyright and licensing header.)  I'm wondering if
these are stripped out before distribution?  It'd help me a lot to extend
code functionality and otherwise fix any encountered problems.

Cheers,

Mark
