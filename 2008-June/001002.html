Hi Frederik, Matt

Sorry for taking a long time to rely. Mapnik has image scaling  
implementation internally :

ImageData32 source(100,100);
ImageData32 target(200,120);
scale_image<ImageData32>(target,source); // stretch source to fit taget

It doesn't use any filtering and visual quality can be sub-optimal but  
it is fast.
For better results I recommend AGG based implementation or  
boost::gil .  I have code somewhere which I happy to share.

Cheers
