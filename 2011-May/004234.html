Hi,

Results from a quick test with four clients. There are some problems with GetCapabiilties and perhaps a bug with format=image/jpeg used together with TRANSPARENT=TRUE but many clients do show a map.  However, Cadcorp SIS Map Browser did not let me go that far.

Kosmo 2.0
- connects
- does not show GetCapabilities
- shows the map with all supported formats (png, png8, jpeg)
- cannot zoom automatically to the data extents, perhaps because I do not know how to do it with Kosmo, or because of faulty GetCapabilites and described later with OpenJUMP

MapInfo 7.5
- connects
- shows map as png, but not as jpeg
- zooms to data extents
It is a long time since I have really used MapInfo and this version (7.5) is pretty old so this was just a preliminary test and should be verified by somebody running a recent Mapinfo version.

Cadcorp SIS Map browser v. 7
- connects
- shows GetCapabilities
- does not go any further, probably because is not satisfied with the information sent by Ogcserver

OpenJUMP
- connects
- cannot zoom into data extents because GetCapabilities do not advertise them correctly for the projected SRIDs and OpenJUMP cannot utilise the lat/lon bounding box;  you can see SRS="UTM +ZONE" in the GetCapabilites
http://terra.girona.cat/ogc/wms/ogcserver_test/?REQUEST=GetCapabilities&SERVICE=WMS&VERSION=1.1.1
- shows map OK with png and png8
- jpeg gives an odd result because OpenJUMP is always sending &TRANSPARENT=TRUE even it does not make sense with format=image/jpeg : compare the following urls with a browser

http://terra.girona.cat/ogc/wms/ogcserver_test/?REQUEST=GetMap&SERVICE=WMS&VERSION=1.1.1&WIDTH=469&HEIGHT=354&LAYERS=municipis&TRANSPARENT=TRUE&FORMAT=image%2Fjpeg&BBOX=236263.13559321914,4476133.050847457,304095.90395480354,4527333.050847459&SRS=EPSG:23031&STYLES=
http://terra.girona.cat/ogc/wms/ogcserver_test/?REQUEST=GetMap&SERVICE=WMS&VERSION=1.1.1&WIDTH=469&HEIGHT=354&LAYERS=municipis&FORMAT=image%2Fjpeg&BBOX=236263.13559321914,4476133.050847457,304095.90395480354,4527333.050847459&SRS=EPSG:23031&STYLES=

Hi All,

I've got a mapnik WMS server setup using WSGI.  The WMS server works fine delivering images, but any XML output from the server includes various 'ns0:' and 'ns1' items (see GetCapabilities example at the end).  These make it appear as a non-valid WMS server to most applications.  Searching the internet turned up a couple of results regarding this that stated all I needed was to have 'SERVICE=WMS' in my request, but it already was in the request.  Any ideas how to get rid of these?

Thanks,
Rob
