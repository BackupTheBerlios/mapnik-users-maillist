>    lyr = Layer(t)
>    lyr.datasource = PostGIS(host=h,port=p,user=u,password = pw, dbname=dbn,table=t)

Hi Ryan,

As long as you pass the 'extent' parameter, the creation on the fly of a PostGIS datasource should be quite fast. So, can you store the extents of your table in your database table and use that when you re-create the Mapnik layer?

If you truly want to stash the list of mapnik postgis parameters then pickling a python dictionary might work nicely.
