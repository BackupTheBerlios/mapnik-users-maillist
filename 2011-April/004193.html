On Sat, 2011-04-23 at 21:16 +0100, David James wrote:
> I'm running Fedora 14.
> 
> I've installed postgresql, postgis and mapnik (0.7.1). All these have
> been installed using yum install with default repositories.
> 
> I can't get mapnik to work with a PostGIS datasource - it says "Could
> not create datasource. No plugin found for type 'postgis'".
> 
> I've googled for this message and tried following the suggestions I've
> found. However, I need some more clues ...
> 
> Most of the articles I've found talk about the postgis.input file being
> missing - there is a postgis.input file (dated Jul 28 2010, size 254556)
> in /usr/lib/mapnik/input
> 
> What do I look for next, please?

The mapnik library needs to be told where to find the plugins and your
application may be telling mapnik to look in the wrong location. What
application are you using? 

Generally the plugin path is configurable in the application, e.g. for
mod_tile / renderd it is this line in the renderd.conf:

  plugins_dir=/usr/local/lib64/mapnik2/input

The latest Mapnik code includes the search path in the error message to
make this more obvious.
